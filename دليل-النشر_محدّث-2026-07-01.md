# دليل نشر التطبيق على الإنترنت

هذا الدليل يشرح كيفية رفع التطبيق ليعمل للعموم. اختر أحد المسارات الثلاثة حسب ما هو متاح لديك.

---

## ما تحتاجه قبل البدء (مشترك)

1. **خادم** يعمل بنظام لينكس (سيرفر الوزارة الحالي، أو خادم افتراضي VPS).
2. **نطاق فرعي** يشير إلى الخادم، مثل `app.awqaf-damas.com` (يُضبط من إعدادات DNS).
3. **سرّ قوي** لمتغير `SECRET` (يُولّد بأمر: `node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"`).

> التطبيق يقرأ ثلاثة متغيرات بيئة: `PORT` (المنفذ)، `SECRET` (سرّ المصادقة)، `DATA_DIR` (مكان حفظ قاعدة البيانات والصور بشكل دائم).

---

## المسار 1 — خادم لينكس / سيرفر الوزارة (موصى به)

أنسب خيار إذا كان لديكم سيرفر خاص.

```bash
# 1) انسخ المجلد إلى الخادم
sudo mkdir -p /opt/awqaf_app /var/lib/awqaf
sudo cp -r awqaf_app/* /opt/awqaf_app/

# 2) تأكد من تثبيت Node.js (الإصدار 18 أو أحدث)
node -v

# 3) ثبّت خدمة التشغيل التلقائي
sudo cp /opt/awqaf_app/deploy/awqaf.service /etc/systemd/system/
sudo nano /etc/systemd/system/awqaf.service   # ضع قيمة SECRET قوية
sudo systemctl daemon-reload
sudo systemctl enable --now awqaf

# 4) جدار عكسي + HTTPS عبر Nginx
sudo apt install nginx certbot python3-certbot-nginx
sudo cp /opt/awqaf_app/deploy/nginx.conf.example /etc/nginx/sites-available/awqaf
sudo ln -s /etc/nginx/sites-available/awqaf /etc/nginx/sites-enabled/
sudo certbot --nginx -d app.awqaf-damas.com
sudo systemctl reload nginx
```

بعدها يعمل التطبيق على `https://app.awqaf-damas.com`. لمتابعة السجل: `journalctl -u awqaf -f`.

---

## المسار 2 — حاوية Docker (يعمل على أي خادم)

إن كان الخادم يدعم Docker:

```bash
cd awqaf_app
docker build -t awqaf-app .
docker run -d --name awqaf -p 3000:3000 \
  -e SECRET="ضع_سرّاً_قوياً" \
  -v awqaf_data:/data \
  --restart unless-stopped awqaf-app
```

ثم تضع Nginx أمامه كما في المسار 1 للحصول على HTTPS.

---

## المسار 3 — منصّة استضافة مُدارة (الأسهل تقنياً)

منصّات مثل Render أو Railway تستضيف تطبيق Node مباشرةً من مستودع Git:

1. ارفع مجلد `awqaf_app` إلى مستودع (GitHub/GitLab).
2. أنشئ خدمة Web Service جديدة، أمر التشغيل: `node server.js`.
3. أضِف متغيرات البيئة: `NODE_ENV=production` و `SECRET=...`.
4. أضِف قرصاً دائماً (Persistent Disk) ووجّه إليه `DATA_DIR` لحفظ البيانات والصور.

> تنبيه: على المنصّات التي لا توفّر قرصاً دائماً، يُفقد ملف `data.json` عند إعادة التشغيل — استخدم قرصاً دائماً أو قاعدة بيانات خارجية.

---

## بعد النشر

- **بوابة SMS:** في الإنتاج لا يظهر رمز الدخول على الشاشة. اربط بوابة رسائل لإرساله فعلياً (موضع الربط معلّم في `server.js` داخل مسار `request-otp`).
- **النسخ الاحتياطي:** انسخ `DATA_DIR` (الذي يحوي `data.json` ومجلد `uploads`) دورياً.
- **التطبيقان للهاتف:** الواجهة تعمل الآن كـ PWA يمكن «تثبيتها» من المتصفح على الهاتف. للنشر على المتجرين، تُغلّف بـ Flutter/WebView (راجع وثيقة المواصفات التقنية).
- **الشعار:** استبدل `public/icon.svg` بالشعار الرسمي الجديد للمديرية.

## سجل التغييرات والمستجدات (2026-07-01)

> هذا القسم يوثّق مزايا وتصحيحات أُضيفت بعد الإصدار الأول، **دون حذف أي محتوى سابق** من هذا الدليل.

### مزايا مضافة تخصّ الإدارة والنشر

- **إدارة الحسابات والمساجد من داخل التطبيق:** صار بإمكان **موظف الوزارة** و**المدير العام** إضافة حسابات القيّمين — وإنشاء مساجد جديدة — مباشرةً من واجهة الويب (قسم «إدارة الحسابات») عبر المسار `POST /api/users`، **دون تحرير `data.json` يدوياً**. القاعدة: **المدير العام وحده** يمكنه إضافة حسابات موظفي الوزارة.
- **التصفّح كزائر (بلا تسجيل دخول):** تتيح الواجهة تصفّح الأغراض المتاحة وقائمة المساجد دون دخول، عبر `GET /api/items?scope=browse` و`GET /api/mosques`. أما كل العمليات (الطلب/الموافقة/التسليم/الإحصاءات) فتبقى محميّة وتتطلب تسجيل الدخول.

### نقاط ضبط مهمّة قبل النشر

- **عنوان السيرفر في تطبيق الموبايل:** القيمة الحالية للمتغيّر `kApiBase` في `mobile_flutter/lib/api.dart` هي **رابط نفق مؤقّت** (من نوع tunnelmole). **استبدله بنطاق المديرية الثابت** (مثل `https://app.awqaf-damas.com`) قبل بناء التطبيق — رابط النفق مؤقّت ولا يصلح للإنتاج.
- **الشعار:** أُضيف الشعار الرسمي فعلاً في تطبيق الموبايل (`mobile_flutter/assets/logo.png`). يبقى فقط استبدال أيقونة الويب `public/icon.svg` بالشعار الرسمي إن رغبت (البند موجود في قائمة التحقق أعلاه).

03 — نموذج البيانات وآلة الحالات

التخزين الحالي في ملف data.json ككائنات JSON. عند الترقية لقاعدة بيانات علائقية (MySQL/PostgreSQL) تتحوّل هذه الكائنات إلى جداول بالعلاقات المبيّنة.

الكيانات

المساجد mosques

الحقل النوع الوصف
id عدد معرّف فريد
name نص اسم المسجد
area نص المنطقة/الحي

المستخدمون users

الحقل النوع الوصف
id عدد معرّف فريد
fullName نص الاسم الكامل
phone نص رقم الجوال (للدخول) — فريد
email نص البريد الإلكتروني (للدخول) — فريد
role نص imam / ministry / manager
mosqueId عدد/فارغ المسجد المرتبط (للقيّم فقط)

الأغراض / العمليات items

في هذا التصميم يُمثَّل الغرض والعملية المرتبطة به في كيان واحد لتبسيط النموذج. عند التوسّع يمكن فصل «طلبات النقل» في جدول مستقل (كما في وثيقة المواصفات).

الحقل النوع الوصف
id عدد معرّف فريد
name نص اسم الغرض
category نص الفئة (تكييف، أثاث، فرش، صوتيات، إنارة، أخرى)
quantity نص الكمية
description نص وصف الحالة
photo نص مسار الصورة (/uploads/...)
sourceMosqueId عدد المسجد المالك/المصدر
status نص الحالة (انظر آلة الحالات)
requesterMosqueId عدد/فارغ المسجد الطالب (عند وجود عملية)
decisionNote نص ملاحظة الموافقة أو سبب الرفض
createdAt وقت تاريخ الرفع/آخر طلب
decidedAt وقت/فارغ تاريخ قرار الوزارة/التسليم

العلاقات

mosques 1 ───< users        (المسجد له عدّة قيّمين محتملين)
mosques 1 ───< items         (sourceMosqueId — المسجد المصدر)
mosques 1 ───< items         (requesterMosqueId — المسجد الطالب)
users   1 ───< items         (قرارات الوزارة — يمكن تسجيل decided_by عند التوسّع)

آلة الحالات (status)

        ┌─ قيّم آخر يطلب الغرض ─┐
[available] ───────────────────► [pending]
     ▲                              │
     │ (الوزارة ترفض)               │ (الوزارة توافق)
     └──────────────┐               ▼
                    └──── [approved] (بانتظار التسليم)
                                    │
                       (المسجد المصدر يؤكّد التسليم)
                                    ▼
                              [delivered]  ← نهائي
الحالة المعنى من يُغيّرها عبر
available معروض ويُطلب القيّم (المصدر) رفع غرض
pending بانتظار موافقة الوزارة قيّم الطالب POST /items/:id/request
approved اعتُمد — قيد التسليم موظف الوزارة POST /requests/:id/approve
rejected* رُفض موظف الوزارة POST /requests/:id/reject
delivered اكتمل النقل قيّم المصدر POST /items/:id/deliver

* عند الرفض يعود الغرض عملياً إلى available ويُلغى الطالب، فيُتاح لمسجد آخر.

قواعد السلامة (Invariants)

  1. لا ينتقل غرض إلى approved/rejected إلا بقرار من دور ministry.
  2. لا يُسلَّم غرض (delivered) إلا إذا كان approved ومن المسجد المصدر.
  3. لا يطلب قيّم غرضاً من مسجده هو.
  4. كل عملية تُعالَج منفردة بقرارها الخاص.

النسخ الاحتياطي

البيانات كلها في DATA_DIR (ملف data.json ومجلد uploads). نسخها دورياً يكفي لاستعادة كامل النظام.

## نظرة عامة على المنظومة

```
        ┌─────────── العملاء ───────────┐
        │  ويب (متصفح/PWA)              │
        │  أندرويد (Flutter)            │
        │  آيفون (Flutter)              │
        └──────────────┬───────────────┘
                       │  REST API + JSON (رمز JWT)
                       ▼
            ┌────────────────────┐
            │  السيرفر (Node.js) │
            │  منطق العمل + الصلاحيات
            └─────────┬──────────┘
            ┌─────────┼──────────┐
            ▼         ▼          ▼
        قاعدة بيانات  صور الأغراض  (مستقبلاً) SMS/إشعارات
```

**القاعدة الجوهرية:** لا تتم أي عملية نقل غرض بين مسجدين إلا بعد موافقة موظف الوزارة عليها منفردةً.

**الأدوار الثلاثة:** قيّم المسجد · موظف الوزارة · المدير العام.

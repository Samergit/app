# 07 — توثيق المحتوى والشاشات (مرجع شامل)

مرجع تفصيلي لكل شاشة وعنصر ونص ورسالة في التطبيق، عبر النسختين (ويب وموبايل) — فالواجهتان متطابقتان في المحتوى. يُستخدم للتدريب، الترجمة، أو ضبط الجودة.

---

## الهوية والنصوص الثابتة

- **اسم التطبيق:** «أوقاف دمشق».
- **العنوان الفرعي:** «تبادل مستلزمات المساجد».
- **الألوان:** أخضر أساسي `#0B6E4F`، ذهبي `#C9A227`.
- **الاتجاه:** من اليمين لليسار (RTL)، اللغة العربية كاملةً.
- **الأيقونة:** مؤقتة (مسجد) — تُستبدل بالشعار الرسمي الجديد للمديرية.

---

## شاشة الدخول

| العنصر | النص/السلوك |
|--------|-------------|
| العنوان | «مديرية أوقاف دمشق» + «تبادل مستلزمات المساجد الفائضة» |
| حقل الرقم | تلميح: «رقم الجوال — مثال 0911000001» |
| زر | «إرسال رمز التحقق» |
| بعد الإرسال | حقل «رمز مكوّن من 6 أرقام» + زر «دخول» + «تغيير الرقم» |
| صندوق تجريبي | يعرض أرقام الحسابات التجريبية |
| رسائل | «أدخل رقم الجوال» · «رمز التحقق (تجريبي): ####» · «رقم الجوال غير مسجّل لدى المديرية» · «رمز التحقق غير صحيح» |

---

## الرأس (بعد الدخول)

اسم التطبيق + اسم المستخدم ودوره (شارة) + زر «خروج».

---

## فئات الأغراض (موحّدة)

| الفئة | الرمز |
|------|------|
| تكييف وتدفئة | ❄️ |
| فرش وسجاد | 🟩 |
| أثاث | 🪑 |
| صوتيات | 🔊 |
| إنارة | 💡 |
| أخرى | 📦 |

## حالات الغرض/العملية (موحّدة)

| الحالة | النص الظاهر | اللون |
|-------|-------------|------|
| available | متاح للطلب | أخضر |
| pending | بانتظار موافقة الوزارة | كهرماني |
| approved | موافَق — بانتظار التسليم | أزرق |
| rejected | مرفوض | أحمر |
| delivered | تم التسليم | رمادي |

---

## شاشات قيّم المسجد

### تصفّح
- عنوان: «أغراض متاحة من مساجد أخرى» + عدّاد.
- بطاقة الغرض: صورة + شارة الحالة + الاسم + «🕌 المسجد» + «🏷️ الفئة · الكمية» + الوصف + زر **«🤲 طلب هذا الغرض»**.
- فارغ: «📭 لا توجد أغراض متاحة».
- رسالة الطلب: «تم إرسال طلبك — بانتظار موافقة الوزارة».

### أغراضي
- عنوان: «أغراض مسجدي المرفوعة» + عدّاد.
- إجراءات حسب الحالة: «سحب الإعلان» (متاح) · «طلب وارد — بانتظار الوزارة» (pending) · «✅ تأكيد التسليم» (approved).
- فارغ: «📦 لم ترفع أي غرض بعد».
- رسائل: «تم سحب الإعلان» · «تم تسجيل التسليم».

### طلباتي
- عنوان: «طلباتي من المساجد الأخرى» + عدّاد.
- بطاقة: الاسم + «المصدر ← مسجدي» + شارة الحالة + ملاحظة الوزارة إن وُجدت.
- فارغ: «📝 لا توجد طلبات».

### رفع غرض (نموذج)
| الحقل | التلميح/الخيارات |
|------|------------------|
| الصورة | «اضغط لالتقاط صورة الغرض» → بعد الالتقاط «تم ✓» |
| الاسم | «اسم الغرض» |
| الفئة | اختيار من الفئات الست |
| الكمية | «الكمية — مثال 1 ، 25» |
| الوصف | «وصف الحالة وسبب الاستغناء» |
| الزر | «رفع الغرض» |
| التحقق | «يرجى التقاط صورة» · «أدخل اسم الغرض» · «اختر الفئة» |
| النجاح | «تم رفع الغرض بنجاح ✓» |

---

## شاشات موظف الوزارة

### الموافقات
- بطاقتان: «بانتظار قرارك» + «موافَق (قيد التسليم)».
- عنوان: «طلبات نقل بانتظار الموافقة — كل عملية تُعتمد منفردة».
- بطاقة العملية: المصدر **←** الطالب + الفئة/الكمية + الوصف + زرّا **«✅ موافقة»** / **«✕ رفض»**.
- فارغ: «✅ لا توجد طلبات معلّقة».
- ورقة الرفض: «رفض العملية» + «سيُبلَّغ المسجد الطالب ويعود الغرض متاحاً» + حقل «سبب الرفض (اختياري)» + «تأكيد الرفض».
- رسائل: «✅ تمت الموافقة على العملية» · «تم رفض العملية».

### السجل
عنوان «سجل العمليات» + عدّاد، وبطاقات كل العمليات بحالاتها وملاحظاتها.

---

## شاشات المدير العام

### اللوحة
عنوان «لوحة الإشراف» + 6 بطاقات (إجمالي / متاحة / بانتظار الوزارة / قيد التسليم / مكتملة / مساجد) + «الفئات تداولاً» كأشرطة نسبية.

### العمليات
نفس سجل العمليات.

### المساجد
عنوان «المساجد المشاركة» + عدّاد، ولكل مسجد: «🕌 الاسم» + «رفع N غرض · طلب M».

---

## الرسائل والأخطاء (موحّدة عبر النسختين)

| الموقف | النص |
|--------|------|
| غير مسجّل دخول | «يجب تسجيل الدخول» |
| صلاحية غير كافية | يُعرض الإجراء معطّلاً أو يُمنع (403 من الخادم) |
| طلب غرض غير متاح | «الغرض غير متاح» |
| طلب غرض المسجد نفسه | «لا يمكن طلب غرض مسجدك» |
| تسليم قبل الموافقة | «لا يمكن التسليم قبل موافقة الوزارة» |

> هذا التوثيق يغطي النسختين الويب والموبايل لأن المحتوى موحّد بينهما؛ أي تعديل نصّي يُطبَّق في الموضعين (`public/index.html` للويب، وملفات `lib/*.dart` للموبايل).
